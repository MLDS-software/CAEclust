Preparing the analysis¶

Loading the training and test datasets¶

CAEclust proposes to easily load two well-known image datasets (USPS and MNIST) for autoencoder-based clustering. Each encoded version of the datasets is processed either with k-means or LSC (Landmark Spectral Clustering).

  • MNIST The database is loaded from the {\tt Keras} Python package. The training and testing sets contain respectively 60,000 and 10,000 images of size 28x28 of the integers in range [0-9]. The images are of grayscale levels rescaled within [0,1] by dividing by 255.
  • USPS The database is prepared as proposed in Guo et al. 2017 and contains 9,298 images of size 16x16 pixels of the 10 digits (integers in range [0-9]) rescaled within [0,1].

Clustering on original datasets¶

We first apply the k-means algorithm on the original training data. We then perform the Landmark-based Spectral Clustering, LSC. This first experiment provides a baseline score.

For LSC, we provide in CAEclust package a Python implementation that follows the Matlab implementation of Chen et al. (Large Scale Spectral Clustering with Landmark-Based Representation, AAAI 2011; Original source: http://www.cad.zju.edu.cn/home/dengcai/Data/code/LSC.m).

Encoding the original data using a DAE¶

For the DAE, we provide the width of three hidden layers for the encoder part (e.g. [500,750,1000]). The decoder stage mirrors the encoder stage. The encoding layer is of size 10.
